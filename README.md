# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

-  リモートリポジトリは、コードを共有したり、共同作業を行うためのネットワーク上に存在するリポジトリ。
- ローカルリポジトリは、コードを編集したり、個人で変更を行うための各々の作業用PCに存在するリポジトリ。

## プッシュとマージの違いは何でしょうか？

- プッシュは、ローカルリポジトリで行った作業をリモートリポジトリに反映させること。
- マージは、異なるブランチで行った作業を統合させること。

## コミットとプッシュの違い

- コミットとは、ローカルリポジトリに何をどう変更したのか記録すること。
- プッシュとは、ローカルリポジトリの変更をリモートリポジトリに反映させること。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

- コミットメッセージはどんな作業を行ったのか正確に書いてあげるのが最適。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

- ローカルでマージするフローは、自分のPC上で直接ブランチを統合すること。
- プルリクエストでマージするフローは、Git上でレビューや確認をしてから統合すること。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

- コンフリクトが発生した場合、以下の３つの方法で対処します。
1. 先にマージされた変更内容を取り込む
2. 後にマージしようとしている変更内容を取り込む
3. どちらの変更内容も取り込む
